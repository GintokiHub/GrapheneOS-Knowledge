Day To Day Usage

"Is it ready to be used as a daily driver OS?"

Resoundingly yes, Graphene OS is stable and feature-rich enough to be utilized as a daily driver operating system with only one consideration: an app store (F-Droid, Yalp, Google Play) is not included by default to allow the user the choice. An app store can be added by simply downloading, verifying and installing the required .apk.

"Do I need to unlock the bootloader to update or flash again?"

No. Your phone will automatically download and install the updates when they are released, provided an internet connection is available. The update images are provided for those who need to download them off of the device for any reason.

Updates are digitally signed by Daniel Micay and verified by the device. Unlocking to update is not needed.

"Can I set the update client to only download on WiFi?"

Yes, under Settings -> System -> Advanced -> System update Settings -> Permitted networks, set it for unmetered networks only to prevent the update client from using up your cellular data plan.

What does the operating system come with?"

"Out of the box", the operating system comes with an app for SMS messages and a Phone app, a search app, Vanadium Browser, Auditor, Mail, Calendar, Photo Gallery, Calculator, Camera, and a File Browser. More apps can always be installed as desired.

"What apps will GrapheneOS run?"

GrapheneOS will run most apps for Android. Most apps from F-Droid will run. GrapheneOS will run proprietary apps such as Spotify and some banking apps if they are loaded from .apk so long as they do not have a hard dependency on the absent Google Software Frameworks.

Apps that are written unsafely or attempt to violate the security policies that GrapheneOS has put in place will be halted immediately, and a notification that the application has stopped working displayed to the user.

"Will Signal Private Messenger work?"

Yes, Signal Private Messenger works great on GrapheneOS.

"Do you support long passphrases on GrapheneOS as an unlock mechanism?"

Yes, long passphrases consisting of many dictionary words (using the "Correct Horse, Battery Staple" approach) work perfectly fine. The limit is 64 characters.

"What security tuning or tweaking does GrapheneOS require off the bat for security or privacy enhancements?"

None. GrapheneOS is designed to be secure by default and requires no tweaking nor custom tuning.

"Do you provide root?"

No, root is not provided. Root is discouraged in a security-focused environment to minimize access, privilege, reduce the attack surface of the applications, and reduce the consequences of user error as much as possible.

"Does GrapheneOS support VPNs and VPN apps?"

Yes. Orbot, OpenVPN for Android, the userspace implementation of WireGuard, and the Private Internet Access VPN app have all been tested and reported working. VPNs can be configured under Settings -> Network & Internet -> Advanced -> VPN.

"I want to use Trezor with my Phone, will it work?"

Yes, Trezor Model Ts are tested and working with GrapheneOS.

"Can I use Yubikeys with GrapheneOS?"

Yes, Yubikeys are tested and working with both NFC and connected by USB.

"Can I use tethering and hotspot?"

Absolutely. Hotspot and tethering have been tested and work fine.

Device support

"Will you support my device?"

Support for other devices is planned, but there are a few requirements for full support from Graphene OS. Rather than aim to try to support many devices, GrapheneOS seeks to support only the devices with the best security, and focus on specific devices to ensure that the most attention can be paid to the quality of the code, review, and testing.

A more detailed answer is in the documentation at the entry under device support.

"Will you support the Pixel 3a?

Yes. The Pixel 3a works fine.

"What will I need to know before I purchase a supported device?"

Devices purchased for running GrapheneOS should be bootloader unlocked. Devices that are sold by Verizon are bootloader locked and cannot be unlocked to allow them to be flashed.

Installing GrapheneOS

"I've flashed the operating system, and now every time my phone boots, it presents me with a yellow screen and warns about my phone having a different operating system installed. Why is this?"

This is normal and expected behaviour, as the phone is now running a different operating system than the factory one installed by the manufacturer. If you see this screen, it is a good sign, as it indicates that the Android Verified Boot is doing its job to verify the software, even though it is different than the factory image.

Device Integrity Monitoring can be used to further attest to the authenticity of the installed operating system.

I've flashed the operating system, and now every time my phone boots, it presents me with an ominous-looking orange screen saying 'software integrity not guaranteed.' What's happening?"

If you see this orange screen, the phone is telling you that the boot loader has been left unlocked. This allows persistent changes to be written to the operating system at will. Unlocking the bootloader is one of the steps taken during installation of the operating system, but it is crucially important that as soon as the installation has been finished, the bootloader is locked again immediately. Your phone should never be used if your bootloader is unlocked; doing so switches off Verified Boot.

Visit the installation guide on locking the bootloader to find out how to fix this issue.

"How difficult is the installation?"

Full factory images are included complete with the flash-all scripts. A step-by-step walkthrough is available on the install guide. You will need to be familiar with the commandline.

"How do I get Android Debug Bridge?"

Linux, Mac, and Windows users should always follow the same process and obtain Android Debug Bridge directly from the official sources. No software needs to be installed, the program simply may be run via the commandline from where it is unzipped to.

Linux users should not utilize the version of Android Debug Bridge packaged by their distro. Not only are these versions often out of date, but often have their versions numbered according to the distro, rather than the official source. Utilizing an out-of-date version of Fastboot may brick your device.

"I'm on Linux. './adb devices' returns a warning about no permissions, or Fastboot hangs at 'Waiting for device'. Help!"

You will need the correct udev rules to allow adb to communicate with the devices or to run adb as root. The workflow for adding the correct adb rules for your device will likely be specific to your distro, so they are not listed here.

When I type in './flash-all' it hangs. What is happening?"

Your /tmp partition may be full or too small. Ensure you see the installation guide.
